# Pocketwise: Executive Summary

## The Question
What should a solo developer build next for a 6-month-old budgeting app with three competing feature requests (investment tracking, bill reminders, shared household budgets) and a millennial user base acquired via Reddit?

## The Answer

**Build bill reminders first (2-4 weeks), then shared household budgets (6-8 weeks). Defer investment tracking.**

## Why This Order

1. **Bill reminders** score highest (4.60/5.00) because they are high-retention, high-feasibility, and deepen your budgeting core. They create a weekly habit loop that keeps users returning. Ship time: 2-4 weeks.

2. **Shared household budgets** score second (3.95/5.00) because they have the strongest growth leverage — every shared budget is a built-in referral. They also lock in couples with doubled switching costs. Ship time: 6-8 weeks.

3. **Investment tracking** scores lowest (1.85/5.00) because it requires external API integrations (Plaid/Yodlee), competes with well-funded incumbents, and drives only quarterly engagement. This is a scope trap for a solo dev.

## The Strategic Principle

**Deepen before you widen.** Bill reminders and shared budgets deepen your budgeting core. Investment tracking widens into a new domain. For a solo dev, depth retains users; width fragments attention.

## Key Metrics to Watch

- **After bill reminders**: DAU/MAU ratio (expect +15-25%), notification open rate
- **After shared budgets**: Invitation acceptance rate, partner retention at 30/60/90 days
- **Always**: NPS, churn rate by cohort, organic acquisition rate from Reddit
