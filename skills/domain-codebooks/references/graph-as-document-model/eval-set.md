# Eval Set: graph-as-document-model

## Should Trigger (10)

1. "Users see layers but underneath it's a node graph — how to bridge this abstraction?"
2. "My NodeNetworkInterface has 200+ methods — is this a God Object?"
3. "How to group multiple graph mutations into a single undo step?"
4. "Adding a new node type requires updating 3 separate registries — how to fix this?"
5. "Should I use a flat element array like Excalidraw or a graph like Graphite?"
6. "How does Krita present its compositing node tree as a layer stack?"
7. "My graph operation finds the wrong node because it assumes specific topology"
8. "What's the right transaction model for graph-based undo? Snapshot vs command vs diff?"
9. "How to map between LayerNodeIdentifier and NodeId without losing information?"
10. "Should node types be registered with macros or loaded as plugins?"

## Should NOT Trigger (10)

1. "How to model relationships in Neo4j?" (graph database)
2. "What's the best schema for my knowledge graph?" (knowledge graph)
3. "How to parse and generate SVG files?" (static file format)
4. "How should I structure my layer panel UI component?" (UI, use userinterface-wiki)
5. "How to implement drag-and-drop reordering in a list?" (UI interaction)
6. "What's the best ORM for my database schema?" (database modeling)
7. "How to implement the visitor pattern in Rust?" (design pattern, not domain-specific)
8. "My editor's message dispatch is too slow" (dispatch, use message-dispatch)
9. "How to implement collaborative editing with CRDTs?" (distributed state, use CRDT codebook)
10. "How to export my document to PDF?" (export, not document model)
