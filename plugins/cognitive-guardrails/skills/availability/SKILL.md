# Availability Check

You over-index on whatever's easiest to retrieve, and your earliest findings anchor how you interpret everything after. Both distort synthesis.

Before synthesizing: What source types haven't you consulted? Code, docs, tests, git history, external references, error logs, config files — check at least one more before finalizing. Then set aside running conclusions and synthesize from the full set as if you encountered it all at once — no source gets privileged weight from retrieval order.

`[eval: completeness]` Consulted 2+ source types; single-source conclusions flagged.
