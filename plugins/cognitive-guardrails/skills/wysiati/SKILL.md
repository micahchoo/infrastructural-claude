# WYSIATI Check (What You See Is All There Is)

You will never spontaneously notice missing information — you'll build the best story from what you have and it will feel complete. That feeling is the bias.

Before proceeding: What information would change your conclusion that you don't currently have? What assumptions haven't you verified? Name two specific things you'd want to check if you had more time. If any are checkable right now, check them before continuing.

Then state what remains unverifiable from code alone — runtime behavior, production load, external service internals, user patterns. These are your blind spots. Name them explicitly so downstream decisions aren't built on silent assumptions.

`[eval: depth]` Check any blind spot verifiable from code/docs before proceeding.
